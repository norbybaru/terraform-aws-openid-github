# Contributing

## Requirements

- Terraform `>= 1.0.10`
- AWS provider `~> 5.0`
- [pre-commit](https://pre-commit.com/)

## Commands

```bash
terraform init       # initialize providers
terraform validate   # validate configuration
terraform fmt        # format all .tf files
terraform plan       # preview changes
```

## Pre-commit hooks

Install the hooks once after cloning:

```bash
pre-commit install
```

The configuration (`.pre-commit-config.yaml`) uses [antonbabenko/pre-commit-terraform](https://github.com/antonbabenko/pre-commit-terraform) (v1.96.1):

- `terraform_fmt` — formats `.tf` files.
- `terraform_validate` — validates the configuration.
- `terraform_docs` — regenerates the reference tables in `README.md`.

> Do not hand-edit the content between the `<!-- BEGIN_TF_DOCS -->` and `<!-- END_TF_DOCS -->` markers in `README.md` — it is regenerated by the `terraform_docs` hook. Update variable/output descriptions in the `.tf` files instead.

Run all hooks manually:

```bash
pre-commit run --all-files
```

## CI

`.github/workflows/terraform.yml` runs on pushes and pull requests to `main`:

1. `terraform fmt -check -recursive`
2. `terraform init -backend=false`
3. `terraform validate`

The workflow fails if formatting is not clean.
