Security Issue Details:
The github_environments variable currently defaults to ["*"], which when combined with
allow_environment in default_conditions, matches ANY environment name in the OIDC sub-claim.
This effectively nullifies environment-based access control since any workflow using any
environment (including ad-hoc ones) can assume the role.

GitHub environment protection rules (required reviewers, wait timers, deployment branches)
are a key security control. The wildcard default bypasses all of these, making
environment-based conditions security theater.

Solution:
1. Change github_environments default from ["*"] to []
2. Add validation block to prevent wildcards (similar to repo variable validation)
3. Document this as a BREAKING CHANGE with migration guidance
